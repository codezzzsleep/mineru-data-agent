# Code of Conduct

This project uses a simple contributor standard:

- Be respectful and specific.
- Critique code, evidence, and claims rather than people.
- Do not share private documents, credentials, or non-public benchmark data without permission.
- Keep competition evidence honest: label synthetic, scripted, cached, or fault-injection artifacts clearly.

Maintainers may edit, hide, or remove comments and issues that violate these expectations.
